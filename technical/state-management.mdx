## Overview

Éxodo implements **vanilla JavaScript state management** without using React, Vue, or any state management library. State is managed through:

- Plain JavaScript objects
- ES6 `Map` data structures
- Event-driven updates
- Direct DOM manipulation

## No Framework Approach

<Warning>
  This application intentionally avoids frameworks to:
  
  - Minimize bundle size (no dependencies)
  - Eliminate build complexity
  - Maximize browser compatibility
  - Reduce cognitive overhead
  - Enable direct HTML/CSS/JS deployment
</Warning>

## Catalog State Object

The main application state lives in `initCatalogAndCart()`:

```javascript app.js:279-293
function initCatalogAndCart() {
  const refs = getCatalogRefs();
  if (!refs) return;

  const state = {
    cart: loadCartFromStorage(),
    activeFilter: null,
    searchQuery: "",
  };
  const savedDeliveryMode = loadDeliveryModeFromStorage();
  setDeliveryModeUi(refs, savedDeliveryMode);

  renderCatalog(refs.productList, state.activeFilter);
  renderCart(refs, state.cart);
  applyCategoryFilter(refs, state, null);
  // ... event listeners
}
```

| Property | Type | Purpose | Initial Value |
|----------|------|---------|---------------|
| `cart` | `Map<string, number>` | Cart items (ID → quantity) | From localStorage |
| `activeFilter` | `string \| null` | Current category filter | `null` |

## Cart State (Map)

The cart uses ES6 `Map` for efficient item management:

```javascript
// Cart structure
Map {
  "cemento-25" => 5,
  "hierro-8" => 12,
  "viguetas-tensolite__3.00" => 8
}
```

### Why Map vs Object?

| Feature | Map | Object |
|---------|-----|--------|
| **Key types** | Any type | String/Symbol only |
| **Iteration** | Preserves insertion order | No guaranteed order |
| **Size** | `.size` property | Manual `Object.keys().length` |
| **Lookup speed** | O(1) | O(1) |
| **Serialization** | Requires `Array.from()` | Direct `JSON.stringify()` |

**Choice Rationale:** Map chosen for clean iteration and deletion APIs.

### Cart Mutations

All cart mutations follow this pattern:

```javascript
function mutateCart(state) {

  // 2. Persist to localStorage

  // 3. Re-render UI
  renderCart(refs, state.cart);

```

## Filter State

### Active Filter

Tracks which product category is currently displayed:

```javascript app.js:625-638
function applyCategoryFilter(refs, state, filter) {
  state.activeFilter = filter || null;
  refs.productSearchInput.value = "";
  state.searchQuery = "";
  renderCatalog(refs.productList, state.activeFilter, state.searchQuery);
  updateProductEmptyState(refs, state.activeFilter, state.searchQuery);
  syncSearchClearButton(refs);
  closeSearchDropdown(refs);
  refs.categoryCards.forEach((card) => {
    const isSelected = card.dataset.filter === state.activeFilter;
    card.classList.toggle("is-active", isSelected);
    card.classList.toggle("active", isSelected);
  });
}
```

**State Changes:**
1. Updates `state.activeFilter`
2. Clears search query
3. Re-renders product list
4. Updates empty state message
5. Highlights active category card

### Filter Values

| Value | Meaning | Products Shown |
|-------|---------|----------------|
| `null` | No filter | None (empty state) |
| `"all"` | Show all | All categories combined |
| `"hierros"` | Hierros category | Just Hierros products |
| `"cementos"` | Cementos category | Just Cementos products |
| ... | Other categories | Respective products |

### Text Normalization

Search is accent-insensitive:

```javascript app.js:923-928
function normalizeText(value) {
  return String(value || "")
    .toLowerCase()
    .normalize("NFD")
    .replace(/[\u0300-\u036f]/g, "");
}
```

**Examples:**
- `"Cañ��" → "cano"`
- `"Eléctrico" → "electrico"`
- `"Cerámica" → "ceramica"`

### Infinite Loop Logic

The slider implements infinite scrolling by cloning slides:

```javascript app.js:692-702
const syncInfinite = () => {
  if (currentIndex >= originals.length + cloneCount) {
    currentIndex -= originals.length;
    updateTrackPosition(false);
  } else if (currentIndex < cloneCount) {
    currentIndex += originals.length;
    updateTrackPosition(false);
  }
  logicalIndex = (currentIndex - cloneCount + originals.length) % originals.length;
  renderDots();
};
```

**State Variables:**
- `currentIndex`: Physical slide position (includes clones)
- `logicalIndex`: Logical position (0 to N-1, for dot indicators)

## Event-Driven Updates

All state changes are event-driven:

```javascript
// Pattern: Event → State Mutation → Re-render
element.addEventListener("event", () => {
  // 1. Mutate state
  state.property = newValue;
  
  // 2. Persist (if needed)
  saveToStorage(state.property);
  
  // 3. Update UI
  renderComponent(state);
});
```

### Event Delegation

Uses event delegation for dynamic content:

```javascript app.js:316-330
refs.cartItems.addEventListener("click", (event) => {
  const target = event.target;
  if (!(target instanceof HTMLElement)) return;

  const action = target.dataset.action;
  const productId = target.dataset.id;
  if (!action || !productId) return;

  if (action === "increase") updateCartQty(state.cart, productId, 1);
  if (action === "decrease") updateCartQty(state.cart, productId, -1);
  if (action === "remove") removeFromCart(state.cart, productId);

  saveCartToStorage(state.cart);
  renderCart(refs, state.cart);
});
```

**Benefits:**
- Single listener for all cart items
- Works with dynamically added items
- Better memory efficiency
- Simpler cleanup

## Render Functions

State changes trigger render functions:

### Cart Render

```javascript app.js:577-623
function renderCart(refs, cartMap) {
  const items = Array.from(cartMap.entries())
    .map(([cartId, qty]) => {
      const product = resolveProductByCartId(cartId);
      if (!product) return null;
      return { ...product, cartId, qty };
    })
    .filter(Boolean);

  if (items.length === 0) {
    refs.cartItems.innerHTML = '<p class="cart-empty">Todavia no agregaste productos.</p>';
  } else {
    refs.cartItems.innerHTML = items
      .map((item) => `
        <article class="cart-item" data-id="${item.cartId}">
          <!-- item markup -->
        </article>
      `)
      .join("");
  }

  const totalItems = items.reduce((sum, item) => sum + item.qty, 0);
  refs.cartTotal.textContent = `Pedido (${totalItems})`;

  if (items.length === 0) {
    refs.cartWhatsapp.classList.add("is-disabled");
    refs.cartWhatsapp.href = CART_WHATSAPP_BASE_URL;
  } else {
    const deliveryMode = getSelectedDeliveryMode(refs.deliveryToggleInput);
    refs.cartWhatsapp.classList.remove("is-disabled");
    refs.cartWhatsapp.href = `${CART_WHATSAPP_BASE_URL}?text=${encodeURIComponent(
      buildWhatsappMessage(items, totalItems, deliveryMode)
    )}`;
  }
}
```

### Catalog Render

```javascript app.js:477-514
function renderCatalog(container, categoryId, searchQuery = "") {
  const normalizedQuery = normalizeText(searchQuery);

  if (!categoryId) {
    container.innerHTML = "";
    return;
  }

  if (categoryId === "all") {
    renderAllCatalog(container, normalizedQuery);
    return;
  }

  const category = productCategories.find((item) => item.id === categoryId);
  if (!category) {
    container.innerHTML = "";
    return;
  }

  const filteredItems = category.items.filter((item) =>
    normalizedQuery ? normalizeText(item.name).includes(normalizedQuery) : true
  );
  const countLabel = normalizedQuery
    ? `${filteredItems.length} de ${category.items.length} productos`
    : `${category.items.length} productos`;

  container.innerHTML = `
    <section class="product-section" data-category="${category.id}">
      <div class="product-section-header">
        <h3 class="product-section-title">${category.title}</h3>
        <span class="product-section-count">${countLabel}</span>
      </div>
      <div class="product-grid">
        ${filteredItems.map((product) => buildProductCardMarkup(product, category.title)).join("")}
      </div>
    </section>
  `;
}
```

## Performance Optimizations

### Debouncing Not Used

<Info>
  Search input is **not debounced** because:
  - Filtering is synchronous and fast (< 1ms)
  - Product catalog is small (< 100 items)
  - Immediate feedback feels responsive
  - No API calls to rate-limit
</Info>

### Efficient Lookups

Product index uses Map for O(1) lookups:

```javascript app.js:1037-1043
function buildProductIndex(categories) {
  const index = new Map();
  categories.forEach((category) => {
    category.items.forEach((item) => index.set(item.id, item));
  });
  return index;
}
```

### Minimal Re-renders

Only affected components re-render on state change:

```javascript
// Example: Search only re-renders catalog, not cart
refs.productSearchInput.addEventListener("input", () => {
  state.searchQuery = refs.productSearchInput.value.trim();
  renderCatalog(refs.productList, state.activeFilter, state.searchQuery); // ← Only catalog
  updateProductEmptyState(refs, state.activeFilter, state.searchQuery);
  syncSearchClearButton(refs);
  renderSearchDropdown(refs, state.searchQuery);
  // renderCart() NOT called - cart unchanged
});
```

## Comparison to Frameworks

| Feature | Éxodo Vanilla JS | React |
|---------|-----------------|-------|
| State container | Plain object | `useState` / Redux |
| Immutability | Manual | Enforced |
| Reactivity | Manual re-render calls | Automatic |
| Component model | Functions + DOM | JSX components |
| Bundle size | 0 KB (no deps) | ~130 KB (min + gzip) |
| Learning curve | JavaScript basics | Framework APIs |
| Browser support | All modern browsers | Requires transpilation |

**Trade-offs Accepted:**
- More verbose update code
- Manual render coordination
- No component lifecycle hooks
- Direct DOM manipulation

**Benefits Gained:**
- Zero dependencies
- No build step
- Maximum browser compatibility
- Simpler deployment
- Full control over updates
