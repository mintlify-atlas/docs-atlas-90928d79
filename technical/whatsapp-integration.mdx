## Overview

Éxodo uses WhatsApp's **URL scheme** (`wa.me`) for all customer communications. No WhatsApp Business API or webhook integration - just simple URL links that open WhatsApp with pre-filled messages.

## Why WhatsApp?

<Info>
  WhatsApp is the dominant communication platform in Argentina and Latin America. Integrating it allows:
  
  - Instant customer contact without forms
  - Real-time conversation for quotes
  - No email delays or missed inquiries
  - Familiar interface for users
  - No backend infrastructure needed
</Info>

## URL Scheme Format

### Basic Structure

```

```

<ParamField path="phone" type="string" required>

<ParamField path="text" type="string">
  URL-encoded message to pre-fill in chat

## Cart WhatsApp Integration

### Build Message Function

```javascript app.js:957-964
function buildWhatsappMessage(items, totalItems, deliveryMode) {
  const deliveryText =
    deliveryMode === "envio" ? "con envio" : "con retiro en el local";
  const lines = [`Hola quiero averiguar estos productos ${deliveryText}:`];
  items.forEach((item) => lines.push(`- ${item.name}: ${item.qty} ${item.unit}`));
  lines.push(`Total de items: ${totalItems}`);
  return lines.join("\n");
}
```

### Message Structure

```text
Hola quiero averiguar estos productos con retiro en el local:
- Cemento x25kg: 5 Bolsa
- Hierro del 8: 12 Barra
- Viguetas tensolite 3.00m: 8 Unidad
- Arena x4mt: 2 Viaje
Total de items: 27
```

**Components:**
1. **Greeting:** "Hola quiero averiguar estos productos"
2. **Delivery mode:** "con envio" or "con retiro en el local"
3. **Product list:** Each item on new line with format `- Name: Qty Unit`
4. **Total count:** "Total de items: N"

### URL Construction

```javascript app.js:618-622
refs.cartWhatsapp.href = `${CART_WHATSAPP_BASE_URL}?text=${encodeURIComponent(
  buildWhatsappMessage(items, totalItems, deliveryMode)
)}`;
```

**Encoding:**
```javascript
const message = "Hola quiero averiguar estos productos con retiro en el local:\n- Cemento x25kg: 5 Bolsa";
const encoded = encodeURIComponent(message);
// "Hola%20quiero%20averiguar%20estos%20productos%20con%20retiro%20en%20el%20local%3A%0A-%20Cemento%20x25kg%3A%205%20Bolsa"
```

### Cart Button States

```javascript app.js:613-622
if (items.length === 0) {
```

- **Empty cart:** Disabled, base URL only

## URL Encoding Reference

### Common Characters

| Character | Encoded | Usage |
|-----------|---------|-------|
| Space | `%20` | Word separator |
| Newline (`\n`) | `%0A` | Line breaks in messages |
| Colon (`:`) | `%3A` | "Product: Quantity" |
| Comma (`,`) | `%2C` | List separators |
| Hyphen (`-`) | `-` | No encoding needed |
| Plus (`+`) | `%2B` | Avoid using `+` for space |

### JavaScript Encoding

```javascript
// Always use encodeURIComponent for message text
const message = "Hola, quiero consultar por caños PVC 110";
const encoded = encodeURIComponent(message);
// "Hola%2C%20quiero%20consultar%20por%20ca%C3%B1os%20PVC%20110"

// For entire URL (if constructing manually)
const fullUrl = `https://wa.me/5493815704653?text=${encoded}`;
```

<Warning>
  **Never use `escape()` or manual encoding** - it doesn't handle Unicode correctly. Always use `encodeURIComponent()`.
</Warning>

## Link Behavior

### Target Blank

All WhatsApp links open in new tab:

```html
<a
  href="https://wa.me/..."
  target="_blank"
  rel="noopener"
>
```

**Security:**
- `target="_blank"`: Opens new tab/window
- `rel="noopener"`: Prevents access to `window.opener` (security best practice)

### Mobile vs Desktop

**Mobile (iOS/Android):**
- Opens WhatsApp app directly
- Message pre-filled in chat
- User just needs to tap "Send"

**Desktop:**
- Opens WhatsApp Web (web.whatsapp.com)
- Requires WhatsApp account linked to phone
- Falls back to app download page if not installed

### Edge Cases

<AccordionGroup>
  <Accordion title="WhatsApp Not Installed">
    User redirected to WhatsApp download page or web.whatsapp.com
  </Accordion>
  
  <Accordion title="Long Messages">
    WhatsApp supports messages up to 65,536 characters. Cart orders unlikely to exceed this.
  </Accordion>
  
  <Accordion title="Special Characters">
    `encodeURIComponent()` handles all Unicode properly (Spanish accents, ñ, etc.)
  </Accordion>

## Best Practices

<Check>
  **DO:**
  - Use `encodeURIComponent()` for all message text
  - Add `target="_blank" rel="noopener"` to all links
  - Include context in messages (product names, dates, etc.)
  - Test on both mobile and desktop
  - Keep messages concise but informative
</Check>

<Warning>
  **DON'T:**
  - Hardcode special characters in URLs
  - Use `+` for spaces (use `%20`)
  - Forget to test with Spanish characters (ñ, á, é, etc.)
  - Send empty or generic messages
  - Use WhatsApp for sensitive data (credit cards, passwords)
</Warning>
