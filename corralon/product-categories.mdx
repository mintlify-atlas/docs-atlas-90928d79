# Product Categories

Éxodo Corralón organizes its inventory into 6 main categories, with over 80 distinct products available.

## Category Structure

The product catalog is defined in `app.js` as a JavaScript array:

```javascript app.js:12-120
const productCategories = [
  {
    id: "hierros",
    title: "Hierros",
    items: [ /* iron/steel products */ ]
  },
  {
    id: "cementos",
    title: "Cementos",
    items: [ /* cement, bricks, aggregates */ ]
  },
  // ... more categories
];
```

## How Categories Are Used

### Category Filtering

When users click a category card, the catalog filters to show only products from that category:

```javascript app.js:635-637
function applyCategoryFilter(refs, state, filter) {
  state.activeFilter = filter || null;
  renderCatalog(refs.productList, state.activeFilter, state.searchQuery);
}
```

### Search Across Categories

The search function looks across all categories:

```javascript app.js:477-498
function renderCatalog(container, categoryId, searchQuery = "") {
  const normalizedQuery = normalizeText(searchQuery);
  
  if (categoryId === "all") {
    renderAllCatalog(container, normalizedQuery);
    return;
  }
  
  const category = productCategories.find(item => item.id === categoryId);
  const filteredItems = category.items.filter((item) =>
    normalizedQuery ? normalizeText(item.name).includes(normalizedQuery) : true
  );
}
```

## Product Data Structure

Each product has this structure:

<ParamField path="id" type="string" required>
  Unique identifier (e.g., "cemento-25", "hierro-8")
</ParamField>

<ParamField path="name" type="string" required>
  Display name (e.g., "Cemento x25kg")
</ParamField>

<ParamField path="unit" type="string" required>
  Unit of measure ("Bolsa", "Unidad", "Kg", "m³", "Viaje")
</ParamField>

<ParamField path="meterOptions" type="string[]">
  Optional: Available lengths for products like viguetas
</ParamField>

<ParamField path="price" type="string">
  Optional: Price to display (currently not used, shows "Consultar")
</ParamField>
