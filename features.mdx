## Overview

The platform combines e-commerce functionality for building materials with service booking capabilities for construction projects. All features are built with vanilla JavaScript and designed for simplicity and reliability.

## Product Catalog Features

### Category System
Products are organized into 6 main categories defined in `app.js:12-120`:

- **Hierros** (13 products): Steel bars, wire mesh, C-profiles
- **Cementos** (22 products): Cement, adhesives, bricks, aggregates
- **Electricidad** (0 products): Placeholder for electrical supplies
- **Sanitarios** (4 products): PVC pipes, fittings, adhesives
- **Chapas** (8 products): Metal sheets, insulation
- **Herramientas** (2 products): Construction hardware

### Search & Discovery

<Steps>
  <Step title="Real-Time Search">
    Users type in the search field and see instant results with highlighted matching text. Search is normalized (accents removed) for better matching.
  </Step>
  
  <Step title="Smart Filtering">
    Click category cards to filter products. The URL hash updates to enable deep linking.
  </Step>

### Product Card Details

Each product displays:

- Unit of measurement (Bolsa, Kg, Unidad, etc.)
- Quantity input field
- Optional meter selection for viguetas (2.00m - 5.60m options)
- "Agregar" button to add to cart

## Shopping Cart Features

### Cart Management

```javascript
// Cart data structure
const cart = new Map([
  ["hierro-8", 5],           // productId: quantity
  ["cemento-25", 10],
  ["viguetas-tensolite__3.20", 12]  // product with meter option
]);
```

<Note>
  Cart persistence uses localStorage with key `exodo_cart_v1`. Data survives page reloads and browser restarts.
</Note>

### Delivery Mode Toggle

Users choose between:
- **Retiro en el local**: Pickup at the store location
- **Con envío**: Delivery to construction site

The toggle is a custom-styled checkbox with smooth animations. Selection is saved in localStorage (`exodo_delivery_mode_v1`).

### WhatsApp Quote Generation

When users click "Enviar lista por WhatsApp", the cart generates a formatted message:

```javascript
function buildWhatsappMessage(items, totalItems, deliveryMode) {
  const deliveryText = 
    deliveryMode === "envio" ? "con envio" : "con retiro en el local";
  const lines = [`Hola quiero averiguar estos productos ${deliveryText}:`];
  items.forEach((item) => lines.push(`- ${item.name}: ${item.qty} ${item.unit}`));
  lines.push(`Total de items: ${totalItems}`);
  return lines.join("\n");
}
```

Example output:
```
Hola quiero averiguar estos productos con envio:
- Hierro del 8: 5 Barra
- Cemento x25kg: 10 Bolsa
- Viguetas tensolite 3.20m: 12 Unidad
Total de items: 27
```

## Business Hours Tracking

### Real-Time Status

The platform shows live open/closed badges for both businesses:

```javascript
function isBusinessOpen(openHour, openMinute, closeHour, closeMinute) {
  const now = new Date();
  const localDate = new Date(
    now.toLocaleString("en-US", { timeZone: "America/Argentina/Buenos_Aires" })
  );
  const currentMinutes = localDate.getHours() * 60 + localDate.getMinutes();
  const opening = openHour * 60 + openMinute;
  const closing = closeHour * 60 + closeMinute;
  return currentMinutes >= opening && currentMinutes < closing;
}
```

## Performance Optimizations

<Steps>
  <Step title="Debounced Search">
    Search input triggers instant UI updates without backend calls
  </Step>
  
  <Step title="LocalStorage Caching">
    Cart and preferences cached locally to reduce data loss
  </Step>
  
  <Step title="Lazy Animations">
    Scroll animations only activate on viewport entry using IntersectionObserver
  </Step>
  
  <Step title="Efficient Re-renders">
    Only affected DOM sections update when data changes (cart, catalog, search results)
  </Step>
</Steps>
